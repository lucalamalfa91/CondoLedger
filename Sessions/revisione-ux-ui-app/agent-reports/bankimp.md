# Cluster BANKIMP — Import banca

## Schermata: Import banca

La subview `data-subview-panel="import-banca"` (index.html:348-368) è un layout a due colonne (`split-layout`): a sinistra upload file + import salvati, a destra anteprima movimenti e coda di associazione manuale. L'etichettatura di scopo è oggi corretta: il tab di sotto-navigazione "Import banca" (index.html:171) è separato dal tab "Import doc." (index.html:170), quindi il problema storico "Import documento raggiungibile solo sotto label Import banca" (rif. R2/UX-005/FLOW-003 nel report precedente) risulta **risolto** — non genera più un finding qui.

Il vero problema del cluster è più a valle: il motore di matching (`js/matching.js`) implementa già tolleranza importo, data-periodo e keyword causale (esattamente ciò che il backlog P-10 chiedeva), ma questi risultati vengono in gran parte **buttati via** nella coda "Da associare manualmente", e l'interfaccia di collegamento manuale ha un default pericoloso. A questo si aggiungono copy non tradotta/non user-friendly nel badge di match, una gerarchia CTA/contenuto invertita e alcuni problemi minori di accessibilità e CSS morto.
