# Cluster DOCIMP — Import documento

## Schermata: Import documento (wizard 4 step: Carica file → Resoconto → Dettaglio → Conferma)

Il wizard vive in `index.html:300-347` (`data-subview-panel="import-doc"`), con logica in `js/main.js` (upload, chiamata AI, conferma) e `js/render.js` (rendering stepper/preview/resoconto), supportato da `document-import-*.js` per schema, hint da nome file, compressione immagini, matching nominativi, mapping verso `dues` e validazione.

**Buone notizie rispetto ai report precedenti (verificate nel codice attuale):**
- **UX-005 / MOB-009 / FLOW-003 (Critical, "import documento nascosto sotto Import banca") risultano RISOLTI**: esiste un tab dedicato "Import doc." separato da "Import banca" (`index.html:170-171`), un CTA primario "Importa documento" nell'hero di Panoramica (`index.html:146`) e la prima voce del FAB "Aggiungi" (`index.html:493`) — raggiungibile in 1-2 tap da Home, non più impilato sotto l'Excel banca.
- **UX-008 (alert nativi)** in questo cluster è **RISOLTO**: il flusso usa `toastError`/`showToast` (`js/toast.js`) e un `confirmDialog` custom basato su `<dialog>` (`js/confirm.js`), non `alert()`/`confirm()` nativi.
- **UX-012 (nessuna CTA dopo import)** è **RISOLTO**: a conferma avvenuta l'app naviga a "Versamenti" e imposta `state.postImportPaymentHint` per un banner guida (`js/main.js:646-651`), anche se il banner vive fuori dalla subview import-doc.

Restano però problemi concreti, alcuni nuovi, alcuni solo parzialmente risolti.

### Note positive aggiuntive (non in tabella, per completezza)
- Feedback duplicato ben gestito: dialog nativo `<dialog>` con 3 scelte chiare (Aggiungi/Sostituisci/Annulla), `index.html:327-337`, `js/main.js:455-475`.
- Compressione automatica delle foto lato client prima dell'upload (`js/document-import-images.js`) riduce concretamente i timeout AI su scansioni da mobile — buona scelta UX invisibile.
- Il loader (`index.html:339-344`) imposta correttamente un'aspettativa temporale ("può richiedere fino a un minuto") nel caso standard, con `aria-live="polite"` e `aria-busy` coerenti — buona base di accessibilità per lo stato di caricamento, salvo il caso di regressione di DOCIMP-001.
