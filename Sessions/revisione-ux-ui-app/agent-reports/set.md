## Schermata: Immobili
Schermata a split-layout (lista case a sinistra, form dati/esercizio/nominativi a destra) raggiungibile da Impostazioni → tab "Immobili" (`index.html` L390, L394-435). Gestita da `renderHousesManageList`, `renderHouseForm`, `renderHouseImportParties` in `js/render.js` e dagli handler in `js/main.js` (creazione/modifica/eliminazione). Coesiste con altri due punti di gestione case: lo `<select id="houseSelect">` in header e il drawer `#houseDrawer` (implementato dopo il report precedente).

### Note di verifica sui riferimenti forniti
- **MOB-004** (report base, non nel report "after-devils-review") riguardava l'header `#houseSelect` (`width: min(160px, 36vw)`) troppo stretto su 390px. Verificato in `css/app.css`: esiste ora un override successivo (`@media max-width:860px { .house-select { flex:1; min-width:0; width:auto; max-width:none; } }`, L536-538) che rende il select flessibile — il troncamento netto a 160px non è più imposto, ma il controllo convive ora con drawer + bottone "+" nello stesso `.house-bar` ristretto (vedi SET-002).
- **FLOW-007** ("Gestione immobili", Medium) chiedeva di implementare il markup `.house-drawer` (allora definito in CSS ma "non usato in HTML") e di consolidare la gestione case in un solo posto. Il drawer è stato implementato (`index.html` L501-509, `js/main.js` L822-831) — target parzialmente raggiunto — ma il vecchio select in header non è stato rimosso e Impostazioni → Immobili resta un terzo percorso completo per lo stesso compito (SET-002).
