# Input operatore — Verifica finale UX/prodotto

## Obiettivo generale

Rendere l'app molto più facile da usare, più chiara, più utile e davvero pronta per essere usata dagli utenti finali.

Analisi in profondità → report strutturato in **italiano**.

## Aree obbligatorie

1. **UX/UI review completa** — chiarezza, gerarchia, leggibilità, coerenza, CTA, feedback, navigazione, empty/error/loading states, a11y base. Per ogni problema: dove, perché, impatto, soluzione concreta.

2. **Revisione mobile approfondita** (priorità assoluta) — app su mobile inutilizzabile: layout, overflow, touch targets, modali/tabelle/form, navigazione, densità, elementi fuori viewport. Proposta mobile-first.

3. **Processi e flussi** — ridondanze, passaggi inutili, sforzo cognitivo. Per ogni flusso: stato attuale, perché funziona male, versione semplificata. Step mancanti o da eliminare.

4. **Miglioramento processo di prodotto** — automazioni, scorciatoie, default intelligenti, progressive disclosure.

5. **Future feature** — nome, problema, valore, priorità (alta/media/bassa), complessità (bassa/media/alta).

6. **Prioritizzazione** — Critical / High / Medium / Low (impatto usabilità, mobile, valore utente).

## Formato output (analysis-report.md)

- **A.** Executive summary
- **B.** Problemi UX/UI (severità + soluzione)
- **C.** Problemi mobile (fix)
- **D.** Processi/flussi da riprogettare
- **E.** Opportunità di miglioramento
- **F.** Future feature roadmap (tabella)
- **G.** Action plan (subito / breve / successivo)

## Linee guida

Review spietata ma utile. Nessun complimento generico. Soluzioni precise. Pattern UX moderni. Preparazione release finale. Ripensare da zero dove necessario.
